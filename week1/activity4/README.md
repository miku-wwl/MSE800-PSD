# Activity 4: Requirements Copy

## Overview

This folder contains a copy of the `requirements.txt` file from `week1/activity3`.

## Files

- `requirements.txt`: A duplicate of the dependencies recorded in `week1/activity3/requirements.txt`.

## Notes

- A new package `numpy` was added to the environment.
- The file was generated using the `pip freeze` command.

## How it was created

1. Activated the conda environment for the project.
2. Installed or ensured `numpy` was present.
3. Ran:
   ```bash
   pip freeze > requirements.txt
   ```
4. Copied the resulting `requirements.txt` output from `activity3` into this `activity4` folder.

## Purpose

This activity demonstrates how to export installed Python packages into a requirements file and keep a copy alongside a related activity.
